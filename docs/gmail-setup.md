# Gmail API Setup

This app sends Gmail messages through the Gmail API using OAuth2 refresh tokens. It does not depend on ChatGPT/Codex having Gmail access.

## Google Cloud Setup

1. Open Google Cloud Console: https://console.cloud.google.com/apis/credentials
2. Create or select a project.
3. Enable the Gmail API for the project.
4. Configure the OAuth consent screen.
5. Create an OAuth Client ID.
6. Add the redirect URI you will use locally. For local manual setup, `urn:ietf:wg:oauth:2.0:oob` may work only for some account/client setups; otherwise use a localhost redirect URI and paste the returned code manually.

## Required Scope

Use the least-privilege Gmail send scope:

```text
https://www.googleapis.com/auth/gmail.send
```

## Local Token Helper

Set these in `.env`:

```env
GMAIL_CLIENT_ID=
GMAIL_CLIENT_SECRET=
GMAIL_REDIRECT_URI=urn:ietf:wg:oauth:2.0:oob
```

Then run:

```powershell
node scripts/gmail-get-token.js
```

Open the printed URL, approve access, then run:

```powershell
node scripts/gmail-get-token.js --code YOUR_AUTH_CODE
```

The helper writes the refresh token to `data/gmail-token.json`, which is ignored by git. Copy the refresh token into the integration credential field `GMAIL_REFRESH_TOKEN`.

## Integration Credentials

For each Gmail-sending integration, configure:

- `GMAIL_USER_EMAIL`: sending Gmail/Google Workspace user.
- `GMAIL_CLIENT_ID`: OAuth client id.
- `GMAIL_CLIENT_SECRET`: OAuth client secret.
- `GMAIL_REFRESH_TOKEN`: refresh token generated by `scripts/gmail-get-token.js`.

Secrets are stored through the app credential store and must not be logged.
