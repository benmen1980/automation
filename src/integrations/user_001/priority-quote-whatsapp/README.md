# Priority Quote Notification to WhatsApp

Webhook integration for user_001. It receives a Priority quote payload, generates the Priority price quotation print URL through Priority Web SDK, and sends a WhatsApp Cloud API template message.

## Webhook

When the local ngrok tunnel is running, the dashboard shows the full public URL:

```text
https://<ngrok-domain>/webhooks/user_001/priority-quote-whatsapp
```

Priority sends the shared verification token in this header:

```text
Priority-BPM-Token: <saved Priority token>
```

The saved token must match the value generated by Priority. In the dashboard this is one visible field named "Priority BPM token". For manual testing, the endpoint also accepts `Authorization: Bearer <saved Priority token>`.

## Payload

```json
{
  "CPROF": {
    "CPROFNUM": "PQ26000001",
    "CDES": "דניאל כהן",
    "ROYY_PHONE": "972507573753"
  }
}
```

Mapping:

- `CPROF.CDES` -> WhatsApp template body parameter 1.
- `CPROF.CPROFNUM` -> WhatsApp template body parameter 2 and the Priority Web SDK `WWWSHOWCPROF` quote lookup.
- Generated Priority print/document URL -> WhatsApp URL button parameter `{{1}}`.
- `CPROF.DES` is accepted as a fallback when `CDES` is missing.

## Credentials

Configure these in the dashboard. Secret values are stored securely and never displayed after saving.

- WhatsApp Access Token
- WhatsApp Phone Number ID
- Template Name
- Template Language Code
- Graph API Version
- WhatsApp Button URL Prefix
- Priority Web SDK URL
- Priority tabula.ini
- Priority Language
- Priority Company
- Priority App Name
- Priority Username
- Priority Password
- Priority Device Name

The Priority Web SDK procedure uses `WWWSHOWCPROF` with language `1` and the Hebrew quote-number sort option `לפי מספר ההצעה`.

The approved WhatsApp template button should use a dynamic website URL such as:

```text
https://priority.simplyct.co.il/netfiles/{{1}}
```

The integration stores that static prefix in `WHATSAPP_BUTTON_URL_PREFIX` and sends only the generated Priority document suffix, for example `1d8021aDA7A8666BF83471D8071CD876BDF17FA.htm`, as the WhatsApp button parameter `{{1}}`.

## Test Modes

- `dry_run`: validates the payload, uses a mock Priority print URL, and shows the request summary without sending.
- `test`: builds the request from saved credentials and a mock Priority print URL without calling Priority or WhatsApp.
- `mock_output`: uses a mock Priority print URL and returns a mock WhatsApp response.
- `live`: calls Priority Web SDK to run `WWWSHOWCPROF`, inserts the returned print/document URL suffix into the WhatsApp URL button parameter `{{1}}`, and sends the real WhatsApp template message.

Logs show the flow directions and summaries:

- JSON from Priority
- Priority print URL prepared before WhatsApp post
- JSON to WhatsApp
- WhatsApp response

Secrets, authorization headers, phone numbers, customer names, and full Priority print URLs must not appear in logs.
