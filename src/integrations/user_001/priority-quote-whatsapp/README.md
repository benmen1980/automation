# Priority Quote Notification to WhatsApp

Webhook integration for user_001. It receives a Priority quote payload and sends a WhatsApp Cloud API template message.

## Webhook

When the local ngrok tunnel is running, the dashboard shows the full public URL:

```text
https://<ngrok-domain>/webhooks/user_001/priority-quote-whatsapp
```

Priority sends the shared verification token in this header:

```text
Priority-BPM-Token: <saved Priority token>
```

The saved token must match the value generated by Priority. In the dashboard this is one visible field named "Priority BPM token". For manual testing, the endpoint also accepts `Authorization: Bearer <saved Priority token>`.

## Payload

```json
{
  "CPROF": {
    "CPROFNUM": "PQ26000001",
    "CDES": "דניאל כהן",
    "ROYY_PHONE": "972507573753"
  }
}
```

Mapping:

- `CPROF.CDES` -> WhatsApp template body parameter 1.
- `CPROF.CPROFNUM` -> WhatsApp template body parameter 2 and URL button parameter.
- `CPROF.DES` is accepted as a fallback when `CDES` is missing.

## Credentials

Configure these in the dashboard. These are integration parameters and remain visible after saving so operators can compare them with Priority and Meta settings.

- WhatsApp Access Token
- WhatsApp Phone Number ID
- Template Name
- Template Language Code
- Graph API Version

## Test Modes

- `dry_run`: validates the payload and shows the request summary without sending.
- `test`: builds the request from saved credentials without sending.
- `mock_output`: returns a mock WhatsApp response.
- `live`: sends the real WhatsApp template message.

Logs show the flow directions and summaries:

- JSON from Priority
- JSON to WhatsApp
- WhatsApp response

Secrets, authorization headers, phone numbers, and customer names must not appear in logs.
